## 2. Real web bug found and fixed (FR-E4-005, SC-E4-005)

During the web leg, unformatted numbers rendered via `double.toString()` diverged
between JS (Dart Web / V8) and the Dart VM:

- Desktop VM: `5.0` (Dart spec)
- JS runtime: `5` (JavaScript double-to-string coercion)

This caused silent formatting differences for any field value that was an
integer-valued finite double (e.g., quantities, unit counts).

**Fix:** a localized `_d()` helper was added to three files, with no-op
behavior on desktop (double VM semantics are already correct):

| File | Change |
|------|--------|
| `lib/src/domain/geometry.dart` | `_d()` helper; integer-valued finite doubles → append `.0` |
| `lib/src/expression/value.dart` | same `_d()` guard on unformatted numeric output |
| `lib/src/rendering/text/text_measurer.dart` | same |

Golden bytes are byte-identical (fix is a no-op on the VM where goldens run).

**Known edge:** `-0.0` renders as `0.0` not `-0.0`. No existing test, golden,
or real report exercises this value; noted for completeness.

**SC-E4-005: MET** — the fix is a localized conditional helper; no engine or
domain change; no public-API change.

test output.

## 7. Print on web (FR-E4-004, SC-E4-002 partial)

The `PrintDialogPresenter` seam routes to `Printing.layoutPdf(...)` on web,
which opens the **browser print dialog** via pdf.js.
`Printing.info().canPrint` returns `true` on web.

**Documented web semantics (at the seam in `jet_report_printer.dart`):**

- The browser shows its native print dialog (not a macOS system dialog).
- User-cancel may not be reported back to the Dart layer — the seam's
  `true`/`false` "handed to OS / cancelled" contract is **best-effort on web**.
- `PrintUnavailableException` remains the path for genuinely unsupported
  environments.
- No seam redesign; the abstraction is sufficient.

## 13. Lessons

- **JS double-to-string diverges from Dart VM for integer-valued doubles.**
  `5.0.toString()` → `"5"` on web, `"5.0"` on VM. Any code path that formats
  doubles to strings without a `NumberFormat` must guard with `_d()` or
  equivalent. The three affected files are `geometry.dart`, `value.dart`,
  `text_measurer.dart`.
- **DDC multi-package chrome tests fail on workspace path resolution.** Run
  `flutter test --platform chrome` per-package, not at the repo root.
- **CanvasKit `toByteData(png)` works without hardening** (Flutter 3.44+;
  CanvasKit is now the only web renderer). The top risk did not materialize.
- **Print cancel-as-success is a known web seam gap.** Document it; don't hide
  it. The `PrintDialogPresenter` abstraction contains the exposure.
